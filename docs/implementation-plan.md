# 实施规划

## 当前状态

当前仓库已经切到“管理端执行真实浏览器抓取，用户端只消费结果”的模式：

1. 管理端负责新增数据源、执行 Playwright 抓取、处理人工验证、AI 分析和审核发布。
2. 用户端读取管理端沉淀后的结果数据，不参与抓取执行。
3. 运行态浏览器文件与平台状态分开保存。

## 已落地

1. 共享状态仓：
   `packages/shared/src/lib/store.ts`
2. 共享发布工作流：
   `packages/shared/src/lib/workflow.ts`
3. 管理端浏览器抓取服务：
   `apps/admin/lib/playwright-crawler.ts`
4. 管理端任务编排：
   `apps/admin/lib/task-service.ts`
5. 管理端 AI 环境变量读取：
   `apps/admin/lib/ai-config.ts`

## 当前数据文件

1. `data/platform-state.json`
   保存数据源、任务、审核记录、发布结果
2. `data/runtime/tasks/<taskId>/`
   保存浏览器抓取的 HTML、文本、截图和 storageState

## 已实现工作流

### 数据源

1. 创建数据源
2. 配置浏览器入口链接
3. 配置等待元素
4. 配置验证方式
5. 配置解析提示

### 抓取任务

1. 从管理端发起浏览器抓取
2. Playwright 打开目标页面并提取 HTML、可见文本和截图
3. 命中登录、验证码、人工验证条件时进入待人工验证
4. 管理员可补充：
   Cookie
   storageState JSON
   验证说明
   通过验证后的页面文本
5. 继续执行后进入 AI 分析

### AI 分析

1. 支持 OpenAI 兼容接口
2. AI 配置来自管理端本地环境变量
3. AI 不可用时自动回退到本地规则解析

### 审核发布

1. 审核页可编辑商品名称、标准类型、价格、库存、置信度和来源片段
2. 可编辑摘要、分析结论、风险提示
3. 发布后自动更新：
   商铺列表
   快照
   差异
   比价结果
   稳定度排行
   总览指标

## 下一步建议

1. 增加站点专属解析器
2. 增加浏览器会话接管或远程人工处理能力
3. 增加发布历史、版本回滚、审计日志
4. 拆分内部状态文件与公开结果文件
